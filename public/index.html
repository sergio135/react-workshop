Virtual DOM

El famoso Dom virtual que otros framework han implementado

¿En que consiste?

El virtual DOM es la generacion exacta del actual DOM
pero guardado en memoria

  • Es una copia exacta
  • Tiene elementos anidados
  • Dispone de sus propios eventos
  • Y demas propiedades
    y metodos del DOM

¿Como Funciona?

Nosotros solo trabajamos con el Virtual DOM

Y cuando el virtual DOM cambia, Actualiza el DOM normal

  • Modelo, DOM Virtual y DOM
  • El modelo actualiza el array, y los childs del DOM cambian
  • El DOM Virtual compara los cambios con la version actual
  • Se actualizan solo los elementos que han cambiado
  • 1 sola llamada al DOM añade el nuevo elemento sin modficar el resto

Ventajas

Solo actualizamos los datos que han cambiado

Mejora de rendimiento

Mejor usabilidad Web

Se encarga de todo el manejo del DOM

No mas watcher sobre el renderizado

Mucho mas sencillo

Componentes React

Abstraccion

¿Que significa esto?

No pueden ser pisadas por otros componentes

No estan accesibles desde la variable window ni la consola

Son compatibles con cualquier desarrollo

¿Como funciona?

La logica JS se encuentra en el servidor

En la primera llamada, el servidor devuelve la ruta solicitada con el JS embebido

Las futuras cargas se producen por AJAX para traer la logica en asincronico

Si volvemos a refrescar el navegador, el servidor vuelve a enviar la ruta con el JS embebido

Metodologia Flux

La verdadera M del MVC

¿Que es Flux?

Es un patron/arquitectura de modelos de datos

Los modelos son independientes y externos a la aplicación

Los datos son inmutables, extraibles, y replegables

No tienen dependencia del famework

No son modelos simulados, es la verdadera M del MCV

Flux en detalle..

Se basa en la creacion de varios contenedores donde guardar los datos

Los contenedores sirven todos los datos dinamicos a la/s aplicacion/es

Para modificar los datos, solo se puede acceder atraves de acciones

Dichas acciones son manejdas por el dispatcher

Los Stores son copiados en cada modificacion, siendo inmutables

Faciles de testear, ejecutar, y traspasar de framework o proyecto

Redux

The Flux Library

¿Que es Redux?

Es una libreria JS que implementa Flux

Extremadamente lijera, 2kB y no tiene dependencias

Puede implementarse en cualquier framework

Nos aporta una serie de metodos para agilizar Flux

Diferencias:

Implementa 1 solo Store, y no soporta Dispatcher

Admite varios reducers que se combinan

Su escalabilidad y testing son increibles
